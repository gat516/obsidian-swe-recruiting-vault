# technical workflow

> use the `pattern add` quickadd command to scaffold notes and problems — it picks the pattern, applies the right template, and links the file back to its hub.

## 1. prime (once per pattern)
- read labuladong chapter
- `pattern add` → note → fills [[labuladong-notes-template]] in `notes/`
- distill into the pattern hub's `## signal` + `## skeleton`

## 2. solve (25–30 min per problem)
- `pattern add` → problem → fills [[problem-template]] in `problems/`
- fill sections 1–4 (ticket), solo
- use ai, solve using claude helping you through logic → fill 5–7
- stuck → peek, then close and re-implement in section 5

## 3. consolidate
- update pattern hub
- add a card to the hub's `## flashcards` section (or to the relevant sub-pattern note)
- bump `attempts +1`
- flip `#wip` → `#done`
- [[07_daily-log]]

## weekly
- re-read touched pattern hubs
- review cards (everything tagged `#flashcard` — hubs and sub-pattern notes)
